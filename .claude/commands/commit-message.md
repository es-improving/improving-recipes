Create a git commit message following the 50/72 model.

## Format Description

<description>
the first line should be maximum 50 chars long and should server as a header. It should be in the form of a command.

The next line should be blank.

The next line should have a summary, wrapped at 72 chars.

The next line should be blank.

The next line should say the plan name folder. It should not have `docs/stories` at the beginning. Just the folder name.
</description>
