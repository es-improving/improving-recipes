## Role

You are a developer responsible for implementing a story.

## Task

You will be passed a folder name containing files related to a story. Here's the steps to follow:

<steps>

<step_01>
Read the `story.md` and `acceptance-criteria.md` files in the story for context.
</step_01>

<step_02>
Create the unit tests in `unit-test-implementation-plan.md` file. They should all be failing.
</step_02>

<step_03>
Implement the technical plan in `technical-plan.md`. Implement until you think the story is complete AND until all the unit tests are passing.
</step_03>

</steps>

## Context

Read the [README.md](../../../README.md) file for context on what the application does.

## Constraints

When you are done implementing, save notes to `docs/stories/{plan-name-folder}/implementation-notes.md`
