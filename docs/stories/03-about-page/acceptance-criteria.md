# Acceptance Criteria: About Page

## Navigation

**Given**: A user is on any page of the application,
**When**: The user looks at the navigation bar,
**Then**: An "About" link is visible in the navigation bar.

**Given**: A user is on any page of the application,
**When**: The user clicks the "About" link in the navigation bar,
**Then**: The user is taken to the About page.

## Page Content — Purpose Statement

**Given**: A user has navigated to the About page,
**When**: The user reads the page,
**Then**: The page clearly states that this application is for demonstration purposes only.

## Page Content — Contact Information

**Given**: A user has navigated to the About page,
**When**: The user reads the page,
**Then**: The presenter's name, "Eric Sowell", is displayed on the page.

**Given**: A user has navigated to the About page,
**When**: The user reads the page,
**Then**: The presenter's email address, "eric.sowell@improving.com", is displayed on the page.

**Given**: A user has navigated to the About page,
**When**: The user clicks the email address,
**Then**: The user's mail client opens a new email addressed to "eric.sowell@improving.com".

## Active Navigation State

**Given**: A user has navigated to the About page,
**When**: The user looks at the navigation bar,
**Then**: The "About" link is visually distinguished as the active/current page (e.g., highlighted or styled differently from the other nav links).
